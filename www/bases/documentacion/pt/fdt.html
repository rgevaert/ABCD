Tabela de definição de campos (FDT) 

 Contém a estrutura de campos da base de dados.

Adicionalmente, às definições do WinISIS, são contemplados os parâmetros adicionais para elaboração das planilhas de entrada de dados segundo os tipos de campos estabelecidos em HTML, edição de subcampos, normalização de terminologia através dos arquivos de autoridade, tesauros ou tabelas de valores (Picklists), apresentação do campo na planilha de entrada de dados, etc. O formato é o seguinte:

Campo (F)
Indica que a entrada corresponde a um campo simples da base de dados.

Subcampo (S)
Esta entrada corresponde à descrição dos subcampos do campo. Deve-se descrever cada um dos subcampos que identificam o campo, com os atributos que lhe correspondem. Com esta informação abre-se, no procedimento de entrada, uma nova janela com os subcampos desagregados, evitando assim a entrada dos delimitadores que devem preceder cada subcampo nas estruturas de dados CDS/Isis. Adicionalmente, é permitido o uso de subcampos repetíveis podendo estes subcampos deslocarem-se dentro da planilha de entrada para que a ordem com que se armazenam facilite desde logo a aplicação da linguagem de formatação dos campos. 

Grupo (T)
Este tipo de entrada é recomendado para facilitar a edição dos campos  que contêm subcampos. Permite editar os subcampos em forma de tabela apresentando separadamente uma folha de entrada para cada subcampo. Após uma entrada do tipo Grupo devem ser definidos os subcampos que compõem o campo.

Etiqueta do registro - formato MARC (LDR)
Este campo contém a informação do cabeçalho no formato Marc, segundo a seguinte especificação:

3005 Estado do registro
3006 Tipo de registro
3007 Nível bibliográfico
3017 Nível de codificação
3018 Descrição da forma de catalogação

O valor do campo 3006 indica o tipo de campo fixo associado ao registo.

A estrutura do cabeçalho fica guardado num arquivo com o nome leader.fdt, armazenado no diretório def da base de dados.

Campo Fixo - formato Marc (M)
Este tipo de campo indica ao gerador de planilhas de entrada de dados que deverá inserir um campo fixo, dependente do tipo de registro selecionado no Cabeçalho. A estrutura dos campos fixos construída segundo uma Tabela de Definição de campos (FDT), a que se acede através da opção Tipos de Registro (Marc) no menu correspondente a Atualizar Definições de Bases de Dados. A opção Tipos de Registro (Marc) aparece no menu de atualização quando é detectada a inserção do campo do tipo LDR na FDT da base de dados

Cabeçalho  (H)
Use esta entrada para colocar os separadores de grupos de campos nos formatos de entrada e saída para facilitar a navegação dentro do formulário de entrada

Linha (L)
Inclui uma linha de separação no formato de entrada. Poderá também conter um URL para fazer referência a uma página que seja necessário acessar no momento da apresentação dos dados.

Copiar documento e estabelecer ligação (U)
Permite copiar um documento para o servidor e incluir uma referência no registro correspondente. Na versão atual do ABCD é obrigatório criar um diretório denominado  bases na raiz do ABCD, contendo um subdiretório com o nome da base de dados onde ficam colocadas as imagens. Este diretório deverá ter permissões de leitura. Outra forma de copiar os arquivos pode ser através do protocolo FTP. O formato de apresentação dos registros deve conter o URL correspondente, para ser possível acessar os objetos referenciados no registro.

Data, hora e operador que cria o registro
Neste campo deve ser indicada a data, hora e nome do usuário correspondente ao operador que cria o registro. Se não forem especificados os subcampos serão utilizados a,b,c respectivamente

Data, hora e operador que  modifica o registro
Neste campo deve ser indicada a data, hora e nome de usuário correspondente ao operador que atualiza o registro. Não sendo especificados os subcampos, serão utilizados a,b,c respectivamente

Identificador único do registro ou Auto-incrementar
Corresponde a um valor numérico sequencial que ABCD incluirá no registro, de forma automática, no momento da sua criação.

 
Etiqueta

Valor numérico que identifica cada campo do registro (tag). O seu valor pode ser desde 001 até 999, com exceção do tipo de campo Leader que aceita etiquetas até 4 dígitos de tamanho.
Título Título ou descrição do campo tal como aparecerá nos formatos de entrada ou saída.

I = Entrada principal

Marque esta entrada se o campo for uma entrada principal do registro. Será mostrada na lista associada ao botão permitindo um acesso rápido a todos os registros.

R = Repetível

 Marque esta entrada se o campo for repetível
subcampos

Indique nesta entrada os subcampos que fazem parte do campo. Pode-se omitir o primeiro subcampo, mas terá de ser colocado o caracter - (travessão) para indicar esta situação.
Pré-Literais

Caracteres de edição para utilizar como pré-literais nos campos da FST e nos formatos de saída. Se o primeiro subcampo do campo não possui delimitador, deve-se colocar nesta posição o caracter - (travessão).

Se este valor estiver presente, ABCD no procedimento de entrada de dados substituirá os delimitadores dos subcampos pelos pré-literais, ao apresentar o formulário de entrada de dados. Quando se guarda o registro os pré-literais serão substituídos de novo pelos delimitadores dos subcampos. Desta forma facilita-se a entrada de dados no caso de campos com um número reduzido de subcampos, por exemplo, um autor pode ser introduzido como Apelido, Nome e ser guardado como Apelido^nNome ou ^aApelido^nNome, consoante a informação fornecida pelos parâmetros de subcampos e Pré-literais
Entrada de dados    
Tipo de entrada


Este valor é usado para criar o formulário de entrada de dados e colocar o comando HTML adequado para a captura do campo. Estão disponíveis as seguintes funcionalidades:

Data (D): Ativa a rotina em javascript para selecionar a data a partir de um calendário.

Selecionar várias Opções (C): Checkbox   
Se for pretendido, pode-se indicar em quantas colunas serão apresentadas as opções, colocando o valor correspondente abaixo de colunas

Rádio (O) - selecionar uma opção : Radio button 
Se for pretendido, pode-se indicar em quantas colunas irão ser apresentadas as opções, colocando o valor correspondente na célula abaixo de colunas

Seleção simples (S): Select (simple)  
Usar quando se pretenda selecionar apenas uma opção entre as disponíveis.

Seleção múltipla (M): Select (multiple)  
Use esta opção quando for possível selecionar mais de um valor de entre os que estão disponíveis. neste caso, o campo deve estar definido como R(epetitivo)

Área HTML(A): Insere um editor HTML para permitir a escrita de texto editado com as opções que são oferecidas pela linguagem HTML

HTML Externo (E):  Insere um editor HTML, mas a informação não é guardada na base de dados mas sim num arquivo externo indicado no registro (por implementar) 

Copiar arquivo (U): Insere um elemento javascript que permite copiar arquivos para o servidor e estabelecer a ligação correspondente

Texto/área de texto (X): Text/Text area: 

Texto/área de texto de tamanho fixo (XF):  Apresenta um Text/Text area com um tamanho fixo do número de caracteres que se podem inserir. O número máximo de caracteres deve ser indicado abaixo de colunas

Tabela (TB): Facilita a edição de campos repetíveis contendo subcampos, apresentando-os em forma de tabela. Adicionalmente, pode-se limitar o número máximo de ocorrências, colocando o valor correspondente abaixo de linhas

Password (P): Preenche o caixa de entrada com * para esconder o conteúdo do campo

 
Linhas

Se o tipo de entrada é T (Grupo), especifica o número máximo de linhas (ocorrências) permitidas para o  campo. Se não for indicado este valor e o respectivo campo tiver zero ocorrências, são apresentadas 10 linhas. De outra forma, aparecem apenas as ocorrências já existentes mais 5 linhas adicionais.

Se o tipo de entrada é X (Text area) ou A (Html area), específica o número máximo de linhas a apresentar na área de texto. Se o tipo de entrada for T, é apresentada uma TextBox. Se o tipo de entrada é T, mas o número de linhas é superior então é apresentada uma área de texto. Se o tipo de entrada é A (Html area), define o tamanho da janela onde aparece o editor HTML.

Se o tipo de entrada é M (Select multiple), específica quantas opções aparecem em lista
Colunas

 

Se o valor de Entrar como é X (Textbox/Text area) ou A (Html area), especifica o número máximo de colunas (caracteres) que aparecem nas linhas da área de texto

Se o valor de Entrar como é T (campos repetíveis com subcampos), o número de colunas determina a quantidade de subcampos definidos.

Se o valor de Entrar como é O (radio button) ou C (checkbox) especifica o número de opções a colocar en cada linha.
Lista de Seleção  
Tipo

Lista controlada a utilizar para gestão da terminologia do campo, segundo as seguintes opções:

D = base de dados
P = pick-list
T = Tesauro
Nome Nome da base de dados a utilizar para o controlo de autoridades, ou do arquivo das opções da pick-list
Prefixo

Se o Tipo de lista de seleção  é D, ou T, indica o prefixo a utilizar para obter os termos da base de dados indicada em Nome. O prefixo aplica-se sobre o arquivo invertido (índice) e através do posting obtém-se a informação diretamente do campo que corresponde ao primeiro registro para o qual aponta o MFN (número interno do registro) do posting.
Exibir como

Formato a utilizar para a lista de seleção do campo do arquivo invertido (índice) através do prefixo indicado. Se a técnica de extração contempla vários campos com etiquetas (tags) diferentes, pode-se indicar o nome do formato a utilizar tanto para a exibição como para a extração. Este formato deve ser elaborado segundo um padrão definido para que seja interpretado corretamente. Veja o exemplo utilizado para a descrição das listas de autoridade da base de dados MARC: @autoridades.pft

Este formato tem a seguinte estrutura:

select e3
        case 1:   v1
        case 100: v100^a,`$$$`v100^a
        case 110: v110^a,`$$$`v110
        case 111: v111^a,`$$$`v111
        case 245: v245^a,`$$$`f(mfn,1,0)
        case 260: v260^a," : "v260^b
        case 270: v270
        case 340: v340
...

...

endsel

A variável associada à opção case do comando select identifica o Id atribuído ao campo na FST; logo especifica-se o formato a utilizar para a descrição do campo na lista de seleção e, em seguida, precedido por `$$$`, o formato a utilizar para apresentar o campo e transferir para planilha de entrada de dados. Se for omitido o formato de extração, é utilizado o formato de exibição como formato de extração.

Em resumo, o processo funciona da seguinte forma:

  1. O prefixo fornecido para acessar o arquivo invertido (índice) e preencher a select box en HTML. Na etiqueta <option> da select box é armazenada a informação do posting.
  2. Quando o utilizador seleciona um termo, transfere-se a informação do posting: Mfn, Id, Número da ocorrência e número de sequência.
  3. Com o MFN acede-se ao registro que contém o termo e ao qual se aplica o formato de extração utilizando o ID  e o Número de ocorrência do posting para localizar no registro o termo solicitado.
    Nota importante: se o campo for repetível é imprescindivel utilizar na FST a pós-literal  % para assegurar que a soma das ocorrências seja guardada como parte do posting.
  4. O termo é extraído do registro, tal como foi introduzido na base de dados, e é colocado na janela correspondente da planilha de entrada de dados.
Extrair como Formato a utilizar para extrair da base de dados, o campo localizado no arquivo invertido (índice) através do prefixo fornecido. Se o formato de exibição é um formato predefinido (@xxxx) e cumpre com a regra estabelecida de separar o formato de exibição do formato de extração com $$$, deve deixar esta entrada em branco
Ajuda

Selecione esta opção se o campo tem um arquivo HTML associado às mensagens de ajuda da entrada de dados. Para construir as mensagens de ajuda use a opção Arquivos de ajuda para os campos da base de dados que faz parte do menu Atualizar a definição da base de dados.

As ajudas são guardadas no diretório bases/dbn/ajudas, onde dbn corresponde ao nome da base de dados

  

 

 

Como usar o editor  (Voltar)

Pode aumentar ou diminuir a dimensão das colunas colocando o curso, na área de títulos da tabela, sobre a linha divisória da coluna cuja dimensão pretende alterar e arrastando essa divisória para a direita ou para a esquerda de forma a ampliar ou reduzir a coluna

Use a tecla [Tab] para navegar através das células

Use as setas de direção [Acima / Abaixo] para navegar através das linhas

Duplo_clic ou [F2] para ativar a edição de uma célula

[Enter] ou   [Escape]   para sair da edição de uma célula

Adicionar uma linha antes   permite adicionar uma linha antes da selecionada

Remover linha selecionada   remove da tabela a linha selecionada

Organizar FDT   permite mover os campos definidos na estrutura (por implementar)

Testar   apresenta um exemplo do formato de entrada que se obtém com a estrutura definida

Listar   apresenta numa página a FDT em uso

Validar verifica a estrutura da FDT e apresenta uma lista dos erros encontrados

Atualizar   valida e guarda a FDT (caso não existam erros)

Cancelar   cancela as alterações efetuadas

 

 

 

Tabelas (picklists) e arquivos de autoridades  (Voltar)

 

Para definir as tabelas ou listas de autoridades associadas aos campos selecione a coluna Lista de seleção-Tipo e tipo de arquivo a utilizar para o controle de autoridades. De seguida deverá clicar sobre browse a fim de obter as possibilidades previstas para cada caso  

Tipo

Lista controlada a utilizar para a manipulação da terminologia do campo, segundo as seguintes opções:

D = base de dados
P = pick-list
T = Tesauro
Nome
Nome da base de dados a utilizar para o controlo de autoridades, ou do arquivo com as opções da pick-list
Prefixo
Se o tipo é D, ou T, indica o prefixo a utilizar para obter os termos da base de dados indicada no authority_name. O prefixo aplica-se sobre o arquivo inverso e através do posting obtém-se a informação diretamente do campo correspondente ao primeiro registro indicado pela componente MFN do posting.
Formato de descrição
Formato a utilizar para a descrição numa lista de seleção, do campo localizado no arquivo invertido, através do prefixo fornecido. Se a técnica de extração contempla vários campos diferentes, pode colocar-se nesta entrada o  nome do formato a utilizar para criar tanto o formato de descrição como o formato de extração. Este formato deve ser elaborado seguindo a regra estabelecida para que possa ser interpretado corretamente.
Por exemplo, consideremos o formato construído para uma base de dados Marc:

select e3
        case 1:   v1
        case 100: v100^a,`$$$`v100^a
        case 110: v110^a,`$$$`v110
        case 111: v111^a,`$$$`v111
        case 245: v245^a,`$$$`f(mfn,1,0)
        case 260: v260^a," : "v260^b
        case 270: v270
        case 340: v340
...
...
endsel

A variável associada à declaração case, identifica:
  • o Id atribuído ao campo na FST
  • o formato a utilizar para a descrição do campo na lista de seleção, e,
  • precedido por `$$$`, o formato a utilizar para transferir o campo da folha de entrada. Desta forma pode-se optar por transferir o campo completo (tal como foi introduzido) ou fazer modificações ao nível do campo ou dos subcampos. Se for omitido o formato de extração, é usado o formato de exibição como formato de extração
Extrair como
Formato a utilizar para extrair da base de dados, o campo localizado no arquivo invertido através do prefixo indicado. Se o formato de descrição é um formato predefinido (@xxxx), deixar esta entrada em branco

Tipos de registro

Se a estrutura da base de dados inclui diferentes tipos de registro, esta situação deve ser definida na estruturada da base de dados no ABCD.

O ABCD Apresenta duas formas diferentes de funcionamento com os tipos de registro:

  1. Para estruturas MARC
  2. Para estruturas diferentes de MARC

Tipos de Registro (estruturas Marc)

Esta opção ajuda definir os tipos de registro que vão funcionar na base de dados quando é utilizado o formato MARC. O ABCD identifica que uma base de dados está estruturada no formato MARC quando na tabela de definição de campos foi incluído o campo tipo LEADER (LDR). A estrutura do campo Leader encontra-se num arquivo denominado leader.fdt, localizado no diretório def da base de dados, e que identifica os seguintes campos:

3005 Status do registro
3006 Tipo de registro
3007 Nivel bibliográfico
3017 Nível de codificação
3018 Descrição da forma de catalogação

O tipo de registro é obtido a partir do campo 3006 do Leader, o qual ajuda a identificar o formato do campo fixo 008 a ser utilizado na descrição bibliográfica do título.

O formato do campo fixo associado a cada tipo de registro fica armazenado numa estrutura similar à da FDT e aparece no procedimento de entrada, dependendo do tipo de registro selecionado no Leader.

Para definir tipos de registro nas estruturas MARC, proceda como se segue:

  1. Ative a opção Tipos de registro: marc no módulo de atualização das definições de bases de dados
  2. Coloque na caixa identificada como Etiqueta 1 a etiqueta do campo do Leader no qual se armazena o tipo de registro (3006). Deixe a caixa correspondente à Etiqueta 2 em branco ou coloque o valor referente ao nível bibliográfico (3007).
  3. Na coluna Planilha de entrada de dados coloque o nome da estrutura que contém a descrição do campo correspondente ao tipo de registro que está descrevendo. Use o link Editar para editar ou criar a estrutura correspondente. Este link abrirá o editor de FDT dado que o campo fixo se mantém numa estrutura similar à da Tabela de Definição de Campos da base de dados
  4. Coloque na coluna correspondente à Etiqueta 1 Valor o valor que pode ter o campo 3006 para lhe associar esse tipo de registo. Deixe a coluna Etiqueta 2 Valor em branco ou coloque o valor referente ao nível bibliográfico (3007).
  5. Coloque na coluna Tipo de registro, Descrição a descrição do tipo de registro, tal como aparecerá no momento de entrada de dados para seleção do tipo correspondente 
  6. Repita os passos 2-6 até esgotar os tipos de registro que deseja definir

Quando terminar clique no botão Atualizar

Os tipos de registro definidos serão armazenados no diretório def da base de dados, sob o idioma ativo no momento da criação, com o nome typeofrecord.tab

Tipos de registro (estruturas diferentes de MARC)

Esta opção ajuda a definir os tipos de registro que vão funcionar na base de dados quando a mesma não utiliza o formato MARC.

Para esse efeito é preciso definir o tipo de registro e atribuir que campos da base de dados (máximo dois) armazenam os valores a serem utilizados para identificar o tipo que corresponde ao registro em processo de criação ou edição de um registro.

Para definir os tipos de registro são necessários 2 componentes:

  1. Identificar o campo (etiqueta) da base de dados a partir da qual será obtido (ou armazenado, no caso de um novo registro) o tipo de registro correspondente à informação que está sendo registrada. A versão atual do ABCD permite definir até dois campos, cujos conteúdos determinarão o tipo de registro e, consequentemente, a planilha de entrada a ser utilizada para a entrada ou edição de registros
  2. Descrever os tipos de registro, tendo em conta os seguintes parâmetros:
  • Nome da tabela de definição (FMT) que contém os elementos a serem utilizados para construir o planilha de entrada de dados
  • Valor que os campos identificados em (1.) contém para que sejam considerados pertencentes a esse tipo de registro
  • Descrição do tipo de registro, a qual aparecerá ao operador no momento de entrada de dados para que proceda à seleção do tipo correspondente

A fim de manter a consistência na parametrização do ABCD, os campos fixos são definidos através de uma FMT que contém a lista dos campos (etiquetas) da  FDT da base de dados que são necessários para construir o formato de entrada. Como só se mantém a etiqueta dos campos, estes herdam automaticamente as características definidas na FDT da base de dados.

Para atribuir os tipos de registro é preciso definir previamente as planilhas de entrada de dados que estarão vinculadas a cada tipo (ver Planilhas de Entrada de Dados (FMT).

Se vai definir tipos de registro para uma base de dados que não segue a norma MARC, faça o seguinte:

  1. Coloque nas caixas identificadas como Etiqueta 1 e Etiqueta 2 as etiquetas dos campos que vão ser utilizados para identificar o tipo de registro. A segunda etiqueta é opcional.
  2. Selecione na coluna Planilha de Entrada de Dados que vai utilizar para entrar o tipo de registro. Esta planilha de entrada deve ser definida previamente com a opção Criar/Planilhas de Entrada de Dados
  3. Coloque o valor que terá o campo identificado em etiqueta1 valor para identificar o tipo de registro
  4. Coloque o valor que terá o campo identificado na etiqueta2 valor para identificar o tipo de registro (opcional)
  5. Coloque na coluna Tipo de registro, Descrição a descrição do tipo de registro, tal como aparecerá no momento de criação para seleção do tipo correspondente
  6. Repita os passos 2-5 tantas vezes quantos forem os tipos de registro que existem ou existirão na base de dados
  7. Quando terminar, clique em Atualizar

Os tipos de registro definidos ficarão guardados no diretório def da base de dados, sob o idioma ativo no momento da criação, com o nome typeofrecord.tab

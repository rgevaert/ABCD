Módulo de Aquisições

O módulo de aquisição se apóia em quatro bases de dados:

  1. Sugestões Contem as solicitações dos usuários quanto à necessidade de aquisição de novos objetos, ou novos exemplares de objetos já existentes. Contém 4 grupos de informações:
    • Dados do objeto solicitado
    • Dados da pessoa ou departamento que solicita o objeto
    • Dados da aprovação ou recusa da solicitação
    • Dados de licitação para a compra do objeto
      • Fornecedor
      • Número da licitação
      • Quantidade
      • Preço unitário
      • Moeda
      • Dados de aceitação da licitação.
         
  2. Ordem de compra
    Armazena a ordem de compra emitida como parte do processo de aquisições. Contém :
    • Fornecedor
    • Número da ordem de compra
    • Data da ordem de compra
    • Dados do objeto adquirido
      • Número da requisição
      • Número da licitação
      • Quantidade
      • Preço unitário
    • Reclamações
  3. Fornecedores Com a informação dos fornecedores que podem participar nos processos de licitação e compra dos objetos
  4. Exemplares
    Contém cada um dos exemplares que foram adquiridos como parte do processo de aquisições.

O processo a seguir é o seguinte:

  1. É criada uma nova solicitação de compra de objeto na base de dados suggestions. Esta solicitação pode referir-se à aquisição de um novo objeto ou à aquisição de novos exemplares de um objeto existente.

    Para cada registro que ingressa na base de dados suggestions é criado de forma automática um campo que contém um número sequencial que é utilizado posteriormente como campo de relação entre as sugestão e as ordem de compra geradas

    Cada objeto na base de dados catalográfica é identificado com um número único que servirá posteriormente para estabelecer a relação entre o objeto e suas copias (exemplares).

    Cada exemplar também receberá um número de Registro/Tombo, o qual será utilizado posteriormente no módulo de circulação e empréstimo.
    Se o objeto já existe na base de dados, então o pedido deve indicar o número atribuído a esse objeto na base de dados catalográfica à qual pertence. Se é um novo objeto, então, será atribuído o número no momento da catalogação do objeto, uma vez finalizada a compra.
  2. Cada sugestão deverá ser aprovada ou recusada. Se é aprovada será necessário indicar o número de exemplares que vão a ser adquiridos
  3. As sugestões aprovadas são submetidas a um processo de licitação, anotando no registro:
    • Fornecedores
    • Número da licitação
    • Data da licitação
    • Quantidade ofertada
    • Preço unitário
    • Moeda
  4. Com a informação da licitação é tomada a decisão de compra e é atualizada na base de dados de sugestões (suggestions) a informação do fornecedor selecionado:
    • Data da aprovação 
    • Opcionalmente são atualizados os campos de quantidade de exemplares e preço unitário
  5. São emitidas as ordens de compra com os fornecedores selecionados. Estas ordens de compra são armazenados na base de dados purchaseorder e são vinculados à base de dados  suggestions mediante o número sequencial criado no momento da entrada da sugestão
  6. Se houver atrasos na chegada do material, então é anotado na base de dados de ordem de compra a data das diferentes reclamações realizadas.
  7. Quando chega o material  adquirido, existem duas opções :
    • Se o objeto for novo, deve ser inserido o número de identificação do objeto, catalogá-lo e criar, na base de  dados copies um registro para cada exemplar, inserindo o seu Número de Registro
    • Se o objeto já existe na base de dados catalográfica, só será acrescentado um novo registro na base de dados copies com o Número de Registro correspondente

Transações

Sugestões 
Use esta opção para inserir na base de dados as solicitações para compra de novos objetos. Sob esta opção poderá criar novas solicitações e indicar quais são aprovadas ou recusadas, colocar as cotações apresentadas pelos fornecedores e indicar o fornecedor cuja cotação foi aprovada

Ordem de compra 
Esta opção permitirá:

  • Gerar a ordem de compra para os fornecedores cuja cotação foi aprovada
  • Inserir na base de dados de sugestões os dados da ordem de compra elaborada para a aquisição do objeto solicitado
  • Revisar as ordens de compra pendentes e inserir as datas das reclamações
  • Eliminar ordem de compra e modificar o estado da sugestão associada
  • Processar a chegada dos objetos adquiridos:
    • Realizar a pré-catalogação (se são novos objetos)
    • Gerar os registros correspondentes aos exemplares adquiridos
    • Disponibilizar os exemplares no sistema de circulação e empréstimo

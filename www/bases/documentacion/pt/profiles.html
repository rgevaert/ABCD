Permissões de acesso aos módulos e funções do ABCD

As permissões são agrupadas sob as seguintes categorias:

1. Permissões relacionadas às bases de dados e formatos de entrada e saída definidas

2. Permissões relacionadas às funcionalidades do ABCD.

Para proporcionar permissões aos usuários são definidos perfis que fazem referência tanto às bases de dados como às funcionalidades às quais tem acesso os usuários pertencentes a cada perfil.

Um perfil pode conter tantas bases de dados conforme seja necessário e possuir acesso a um ou mais módulos.  Neste último caso será apresentado no menu principal um submenu para que o operador possa navegar através dos módulos selecionados.

Uma vez construídos os perfis, deve-se entrar nos registros dos usuários e lhes atribuir o perfil corresponde às suas necessidades.

Uma vez criados, os perfis podem ser editados ou eliminados. No entanto, se um perfil está atribuído a um usuário ou mais usuários, o mesmo não poderá ser eliminado até que exclua as relações dos usuários existentes para este perfil.

ABCD possui um grupo denominado adm o qual corresponde ao administrador do sistema. Não é permitido editar ou excluir o perfil deste grupo e pelo menos um usuário deve estar atribuído ao mesmo. De outra forma, corre-se o risco de desabilitar o acesso a funções críticas do sistema.

Se o administrador do sistema for removido por engano, siga o seguinte procedimento para atribuir um administrador provisório para entrar no ABCD e definir o novo administrador:

  1. Edite o arquivo config.php presente na pasta central e coloque um valor provisório nos parâmetros $adm_login e  $adm_password. Este valor provisório não deve corresponder a nenhuma das identificações de usuário já existentes na base de dados de controle de acesso

  2. Entre no ABCD e utilize como identificação do usuário e senha os valores inseridos em $adm_login e  $adm_password, respectivamente

  3. ABCD proverá acesso com o perfil do administrador do sistema. Veja então à opção Administração de usuários e restitua o administrador do sistema.

  4. Apague login e senha provisórios de config.php

Os perfis de usuário são armazenados na pasta par/profiles. Nela existe um arquivo para cada perfil criado. O arquivo profiles.tab  é interno do ABCD e provê os parâmetros que podem ser utilizados para restringir/acessar as funcionalidades do ABCD. Não se deve apagar já que é a matriz utilizada para a criação/edição de perfis.


O arquivo adm contém o perfil do administrador do sistema e não pode ser editado nem apagado. Por esta razão não é apresentado na lista de perfis ativos.

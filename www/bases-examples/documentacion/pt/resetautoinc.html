Restaurar o Número de Tombo/Registro ou o Número de Controle de uma base de dados

Número de Tombo/Registro

O número de tombo é uma numeração, que pode ser criada automaticamente no ABCD para os exemplares (cópias). Este número está definido na FDT da base de dados com o tipo de campo Autoincrementar e é incrementado a cada registro criado, tomando o último valor armazenado e somando 1.

O último valor armazenado é obtido de um arquivo denominado control_number.cn localizado na pasta data da base de dados em questão.

Esta opção modifica o valor do último número atribuído com o objetivo de restabelecer a numeração dos registros.

Número de Controle

O número de controle é um campo obrigatório nas bases de dados bibliográficas cujos exemplares serão manipulados através da base de dados copies, por ser o campo de relação entre o objeto e os seus itens físicos (exemplares).

Como no caso anterior, ABCD pode gerar de forma automática este número de controle através de uma numeração sequencial. O último número atribuído é mantido no arquivo control_number.cn localizado na pasta data da base de dados em questão.

Esta opção modifica o valor do último número atribuído com o objetivo de restabelecer o valor do número de controle dos registros.

Caso queira normalizar o número de controles de forma que em todos os registros tenha o mesmo tamanho em bytes, o mesmo pode ser restabelecido com um valor de acordo com o tamanho previstro para a base de dados. Por exemplo: se a base de dados terá no máximo 89.999 registros, pode-se iniciar ou restabelecer o número de controle para 10000. Assim o primeiro será 10001, o segundo 10002, o terceiro 10003 e assim por diante.até 99999 todos os números de controle terão 5 dígitos. Considerando que o valor atribúido para o primeiro registro foi 100000, esse exemplo permite gerar no máximo 89999 registros com número de controle de 5 dígitos.

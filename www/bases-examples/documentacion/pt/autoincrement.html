Geração automática do número de controle dos registros de uma base de dados


Esta opção permite atribuir de forma automática um número sequencial consecutivo aos registros de uma base de dados.

Na FDT, defina o campo de controle como tipo de campo Autoincrementar. Na definição do campo inclua o prefixo CN_ com o qual o dado deve ser indexado no arquivo invertido e nos formatos de exibição e extração correspondentes.

A numeração está no arquivo control_number.cn, armazenado na pasta data da base de dados correspondente. Neste arquivo é armazenado o último número atribuído no processo de entrada de dados.

O número de controle não pode ser inserido diretamente, no formulário de entrada de dados. A atribuição do número pode ser feita por um dos seguintes métodos:

  1. Se deixar o campo em branco, no momento de armazenar o registro, é determinado o próximo número a ser atribuído e o mesmo é gravado no campo correspondente
  2. Para atribuir um número de controle diretamente no registro, executar os seguintes procedimentos:
    • Clique no link Atribuir que é exibido ao lado do campo Número de controle
    • É exibida uma nova janela para que se digite o novo número a ser atribuído
    • Clique em Enviar para verificar se o novo número de controle não existe
    • Se o número de controle ainda não existe, o valor é inserido no campo
    • Se o número de controle já existe na base, na mesma janela é exibido o registro do documento que tem o número. Clique em Voltar para atribuir outro número ou em Fechar para fechar a janela
  3. Para deixar o campo de número de controle em branco, clique em Atribuir e na janela aberta, deixe a caixa do número em branco, e em seguida clique em Enviar.

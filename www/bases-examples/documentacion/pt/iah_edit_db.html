Editar o arquivo DBN.def para configurar uma base de dados para o iAH 

Esta página de configuração consta de 4 seções necessárias para configurar uma base de dados no iAH:

  • FILE_LOCATION
  • INDEX_DEFINITON
  • APPLY_GIZMO
  • FORMAT_NAME
  • HELP_FORM
  • PREFERENCES

Para informação específica relativa às seções deverá ser consultado o Manual do Usuário do iAH. Nesta página apenas é fornecida uma ajuda sobre a forma de utilizar a ferramenta de configuração prevista no ABCD e não sobre o conteúdo dessas seções.

Para editar o arquivo de configuração são possíveis 2 opções:

  1. Usar um editor TXT para modificar diretamente o arquivo de configuração. Para tal clicar sobre a ligação Editar no formato TXT que é apresentado no início da página.
  2. Usar o assistente fornecido pelo ABCD para facilitar o preenchimento das seções.

O assistente está dividido nas 6 seções exigidas. Cada uma possui no final um link Adicionar  para permitir adicionar um novo elemento da seção em alteração.

A seção  INDEX_DEFINITON permite editar cada linha com vista a facilitar o preenchimento da informação. Para tal clicar sobre editar que acompanha a definição de cada índice e no formulário apresentado e forneça a informação correspondente. Uma vez preenchido o formulário clicar sobre Aceitar para transferir as caixas da linha correspondente da seção INDEX_DEFINITION.

A seção FORMAT_NAME  facilita o acesso ao editor dos formatos de descrição (PFT`s) do ABCD para que seja possível criar ou alterar o formato correspondente.  Este acesso é feito através do link Criar/editar formato incluído no inicio da seção. Se tiver criado um novo formato, deverá adicionar o mesmo à seção, preenchendo as caixas que são apresentadas depois de clicar em Adicionar.

Uma vez preenchida toda a informação no formulário, clicar sobre Validar para verificar a consistência da informação fornecida e depois sobre Atualizar para guardar o arquivo criado.

Os critérios de validação utilizados são os seguintes:

FILE_LOCATION

  1. Verificação de que todos os nomes lógicos estão precedidos pelo literal FILE. Em caso de falta deste literal o mesmo será adicionado.
  2. Verificação de que todos os caminhos estão precedidos pelo literal %path_database%. Em caso de falta deste literal o mesmo será adicionado.

INDEX_DEFINITION

  1. Verificação de que todos os nomes lógicos estão precedidos pelo literal INDEX. Em caso de falta deste literal o mesmo será adicionado.
  2. Verificação de que está presente pelo menos um sub-campo com o nome do índice
  3. Verificação de que os nomes lógicos das bases de dados de índices (^y) tenham uma entrada correspondente na seção FILE_LOCATION

APPLY_GIZMO

  1. Verificação de que os nomes dos gizmo's fornecidos tenham uma entrada correspondentes na seção FILE_LOCATION

PREFERENCES

  1. Verificação de que está ativado pelo menos um formulário de pesquisa.
  2. Verificação de que o número de documentos por página é um valor numérico

Módulo de Empréstimo

Este módulo trabalha com as seguintes bases de dados:

users
Base de dados de usuários de empréstimo
loanobjects

Base de dados com os diferentes exemplares (itens) dos objetos que estão disponíveis para empréstimo.

Existe uma única base de dados loanobjects que reflete o acervo de todos os catálogos definidos no ABCD para empréstimo. Cada item é identificado através de um número único ou número de tombo. Além disso, armazena o número de controle do objeto, o volume e o número de registro do item para estabelecer as diferenças entre os itens de um mesmo objeto, o nome da base de dados onde está catalogado para identificação do item e o tipo de objeto atribuído ao item para efeitos de aplicação da política de empréstimos.

Cada objeto representa um registro da base de dados loanobjects e as cópias (exemplares) do mesmo são armazenadas como ocorrências de um campo repetitivo.

Esta base de dados é atualizada a partir da catalogação, onde é permitido transferir os registros de exemplares/itens da base de dados copies para a base de dados loanobjetcs.

Base de dados do catálogo (cepal, marc, etc)
Contém a descrição bibliográfica dos objetos disponíveis para empréstimo. A ligação entre o catálogo e seus itens físicos é realizada através do número de controle do objeto.
trans
Transações de empréstimos, renovações e devoluções
suspml
Suspensões e multas registradas em relação aos usuários
reserva 
Reservas solicitas

O menu principal do modulo de Circulação e Empréstimo contém as seguintes seções:

Transações

Esta seção define as transações disponíveis:
  • Emprestar
    Através do código do usuário e do número de tombo ou código de barras se acessa as bases de dados users e loanobjects para determinar o tipo de usuário e o tipo de objeto. Além disso, é verificado se o usuário tem multas ou suspensões pendentes, realizando uma busca na base de dados suspml, ou se tem empréstimos pendentes, realizando uma busca na base de dados trans. Com estes valores é determinada a política de empréstimo a ser aplicada e calculada a data de devolução
  • Devolver
    é utilizado o número de tombo ou código de barras para localizar na base de dados trans o exemplar que se deseja devolver. Se há atraso na data de devolução, é calculada a sanção que é indicada pela política de empréstimos e a mesma é salva na base de dados suspml
  • Renovar
    é utilizado o número de     tombo ou código de barras para localizar na base de dados trans o exemplar que se deseja renovar. Além disso, é verificado se o usuário tem multas e/ou suspensões pendentes, realizando uma busca na base de dados suspml, ou se tem outros empréstimos pendentes, realizando uma busca na base de dados trans. Com estes valores se determina a política de renovação a aplicar e se calcula a nova data de devolução
  • Suspender/Multar
    Indica-se o código do usuário que se deseja suspender/multar e os dados correspondentes.
  • Estado da conta de um usuário
    Exibe a listagem de empréstimos, suspensões e/ou multas pendentes para um usuário
  • Situação de um objeto
    Exibe todos os exemplares (itens) de um objeto, indicando aqueles que estão em poder de algum usuário
